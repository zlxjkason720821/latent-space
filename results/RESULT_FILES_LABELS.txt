# Results folder — filename labels (English)

## 1. Evaluation (from: py scripts/evaluate.py)

  report.txt
  → Text report: latent dimension count, action count, which latent index best tracks Mario height, and R² of height-tracking. One-line summary for pipeline metrics.

  rollout_comparison.png
  → Image: side-by-side comparison of real vs predicted rollouts (if generated by evaluate script).

  stability.png
  → Image: latent rollout stability or error-over-steps plot (if generated).


## 2. Physics blending (from: py scripts/physics_blending.py)

  (Any .png written here by physics_blending.py)
  → Blending or interpolation comparison between learned dynamics and reference (exact naming depends on script).


## 3. Play in latent — per-run outputs (from: play_in_latent.py or play_*.bat)

  play_frames/
  play_frames_right/
  play_frames_jump/
  play_frames_mixed/
  play_frames_calm/
  play_frames_<tag>/
  → Folder: decoded predicted frames for one latent-space rollout. Each frame is one timestep; no real game, only dynamics + decoder.

    frame_000.png, frame_001.png, frame_002.png, ...
    → Single predicted game frame at step 0, 1, 2, ... (3-digit zero-padded index).

  play_in_latent_strip.png
  play_in_latent_strip_right.png
  play_in_latent_strip_jump.png
  play_in_latent_strip_mixed.png
  play_in_latent_strip_calm.png
  play_in_latent_strip_<tag>.png
  → Single image: horizontal strip of all predicted frames for that run (timeline left → right).

  latent_height_right.png
  latent_height_jump.png
  latent_height_mixed.png
  latent_height_calm.png
  latent_height_<tag>.png
  → Plot: value of the “height-tracking” latent dimension over rollout steps. Rises when JUMP or RIGHT+JUMP is applied (orange band), then falls (parabola); used to show that the model captures jump physics.


## 4. Tag summary

  right  = move right only.
  jump   = jump (and right).
  mixed  = mixed actions (right, jump, right+jump).
  calm   = slower / fewer actions.
  <tag>  = custom tag from: py scripts/play_in_latent.py --tag <tag> ...


End of labels.
